# Yaga Calls: 30-Day Indexing & Entity Monitoring Protocol

This protocol defines the weekly tasks to ensure successful P0/P0.5 rollout.

## Day 0: Baseline & Submission
*   [ ] Verify `sitemap.xml` and `robots.txt`.
*   [ ] Submit sitemap to Google Search Console and Bing Webmaster Tools.
*   [ ] Manually inspect Top 5 priority URLs in GSC (Home, About, Review, Trading Group, Telegram Group).
*   [ ] Request indexing for Top 5 priority URLs.
*   [ ] Document baseline impressions/clicks for "Yaga Calls" brand.

## Day 3: Discovery Check
*   [ ] Check GSC Sitemap status (Success/Errors).
*   [ ] Check "URL Inspection" for Top 5 URLs to see if crawled.
*   [ ] Verify schema detection in GSC (Organization, FAQ, Breadcrumb).

## Day 7: First Performance Review
*   [ ] Audit GSC "Performance" report.
*   [ ] Check if brand aliases (Yagacall, Yaga) are appearing in the query list.
*   [ ] Verify if `/crypto-trading-group` is starting to show impressions.

## Day 14: Canonical & Authority Check
*   [ ] Check if Google has selected the correct canonical for `/about-yaga-calls` vs `/yaga-calls-review`.
*   [ ] Check for "Crawled - currently not indexed" status in GSC.
*   [ ] Submit remaining P0 URLs to Bing Webmaster Tools.

## Day 21: Cannibalization Audit
*   [ ] Compare query-to-page mapping.
*   [ ] Check if `/telegram-crypto-signals` is conflicting with `/crypto-trading-telegram-group`.
*   [ ] Strengthen internal links to the preferred ranking page if conflicts occur.

## Day 30: Monthly Entity Report
*   [ ] Calculate total Brand vs Category impressions.
*   [ ] Identify "Top 10" queries by impression volume.
*   [ ] Prioritize P1 content expansion or P0 page optimizations.
